# 📄 1. 데이터베이스 정의서 (Database Schema)
# 📊 Database Schema

## 1. rooms 테이블

| 컬럼명             | 타입            | 제약조건               | 설명                                        |
| --------------- | ------------- | ------------------ | ----------------------------------------- |
| id              | BIGINT        | PK, AUTO_INCREMENT | 룸 ID                                      |
| name            | VARCHAR(100)  | NOT NULL           | 룸 이름                                      |
| room_type       | VARCHAR(50)   | NOT NULL           | 룸 타입 (STANDARD, DELUXE, SUITE, PENTHOUSE) |
| capacity        | INT           | NOT NULL           | 최대 수용 인원                                  |
| price_per_night | DECIMAL(10,2) | NOT NULL           | 1박 가격                                     |
| description     | VARCHAR(500)  | NULL               | 룸 설명                                      |
| available       | BOOLEAN       | NOT NULL           | 운영 가능 여부                                  |
| image_url       | VARCHAR(255)  | NULL               | 이미지 URL                                   |

---

## 2. reservations 테이블

| 컬럼명              | 타입            | 제약조건                    | 설명      |
| ---------------- | ------------- | ----------------------- | ------- |
| id               | BIGINT        | PK, AUTO_INCREMENT      | 예약 ID   |
| room_id          | BIGINT        | FK (rooms.id), NOT NULL | 룸 ID    |
| guest_name       | VARCHAR(100)  | NOT NULL                | 고객 이름   |
| guest_email      | VARCHAR(100)  | NOT NULL                | 고객 이메일  |
| guest_phone      | VARCHAR(20)   | NOT NULL                | 고객 전화번호 |
| check_in_date    | DATE          | NOT NULL                | 체크인 날짜  |
| check_out_date   | DATE          | NOT NULL                | 체크아웃 날짜 |
| number_of_guests | INT           | NOT NULL                | 인원 수    |
| total_price      | DECIMAL(10,2) | NOT NULL                | 총 가격    |
| status           | VARCHAR(20)   | NOT NULL                | 예약 상태   |
| special_requests | VARCHAR(500)  | NULL                    | 요청사항    |
| created_at       | DATETIME      | NOT NULL                | 생성일     |
| updated_at       | DATETIME      | NULL                    | 수정일     |

---

## 3. 관계

* rooms (1) : reservations (N)
* 하나의 룸은 여러 예약을 가질 수 있음

# 📄 2. API 정의서 (Function API)
# 📡 Reservation API

## 1. 예약 생성

* **POST** `/api/reservations`

### Request

```json
{
  "roomId": 1,
  "guestName": "홍길동",
  "guestEmail": "test@example.com",
  "guestPhone": "010-1234-5678",
  "checkInDate": "2026-05-01",
  "checkOutDate": "2026-05-03",
  "numberOfGuests": 2,
  "specialRequests": "조용한 방 부탁드립니다"
}
```

### Response (201)

```json
{
  "id": 1,
  "status": "PENDING",
  "totalPrice": 200000
}
```

---

## 2. 내 예약 조회

* **GET** `/api/reservations/my?email={email}`

### Response

```json
[
  {
    "id": 1,
    "roomName": "디럭스룸",
    "checkInDate": "2026-05-01",
    "checkOutDate": "2026-05-03",
    "status": "CONFIRMED"
  }
]
```

---

## 3. 예약 취소

* **PATCH** `/api/reservations/{id}/cancel`

### Response

```json
"예약이 취소되었습니다."
```
